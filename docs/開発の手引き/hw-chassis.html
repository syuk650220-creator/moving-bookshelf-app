メカナム筐体の発注・組立

メカナムホイールの走行筐体（台車）を発注・組立。4輪が付き、ローラーが配置図どおりの向きで、本棚相当の重さを載せても歪まずガタつかず倒れない台車を作る。

このIssueは何？

ゴールは、移動本棚ロボの土台になる走行台車（筐体）を発注して組み立てること。メカナムホイールという特殊な車輪を4つ付けた台車で、後の別Issue（駆動・走行制御）でモータをつなぐと前後・左右・斜め・その場回転の全方向移動ができるようになる。このIssueでは『動力なしの状態で、丈夫でガタつかない台車を物理的に完成させる』ところまでをやる。

なぜ必要か: 本棚（5段＋本）を載せて研究室を自走させるロボなので、まず重い本棚を支えて運べる頑丈な足回りが無いと何も始まらない。台車の強度・サイズ・重心がここで決まり、以降の機構（本棚の載せ方）・駆動（モータ）・ナビ（Nav2）の全部が、この台車を前提に組み上がる。だからSprint序盤に着手する。

これは物理作業（VS Code 無関係）

コードは書かない。発注・組立・測定が中心の物理Issue。ただし『進め方＋調べ方＋安全注意』はアプリIssueと同じ温度感で丁寧に。手を動かす前にまず調べて見積もるのがコツ。

全体の中での位置づけ（疎結合とは別の話）

アプリ⇄ロボはSupabase経由の疎結合だが、それはソフトの話。この台車はロボの『体』そのもの。台車の物理仕様（サイズ・耐荷重・モータ軸径）は、後段のハードIssue（駆動・本棚機構）と物理的に直結するので、関連Issue担当との事前すり合わせが超重要。

完了の定義（Done）

『丈夫そう』ではなく手で確かめられる基準で判定する。完全な数値計測が無理でも、下の『具体的なチェック動作』ができればOKとする。

  • メカナム4輪が、付属の配置図どおりの向き（ローラー軸の傾きが対角で揃う）でフレームに固定されている
  • ステップ1で見積もった『載せる予定の重さ（本＋棚＋電装の合計）』相当の荷物を実際に載せても、フレームの目視たわみが数mm以内で、外したらすぐ元に戻る（永久変形しない）
  • 荷物を載せた状態で台車を手で軽く揺すっても、ネジの緩みによるガタつき音・ぐらつきが無い
  • 荷物を載せて静止させ手を離しても、自然に倒れない（重心が安定している。走行中の安定性は駆動Issueで確認）
  • 前に押すと真っ直ぐ転がる。各ホイールの外周ローラーを指で回すと、4輪とも斜め方向に自由に回る（引っかからない）
  • 選定したモータの軸径・取付穴ピッチが台車側と合う（モータ込みセットキットならこの項目は自動で満たされる）
  • PMに見積りを出し、発注承認を得た上で発注・受領している
横移動・その場回転はここでは判定しない

メカナムの横スライド・斜行・その場回転は『4輪を別々の向き・速度で動力で回して』初めて成立する。電源OFFの手押しでは正しく組めていても横にはスライドしない（ローラーは受動で自由回転するだけ）。全方向移動の確認は、モータをつなぐ後段Issue（駆動・走行制御）で行う。ここで横に動かなくても組み間違いとは限らない。

進め方ステップ

  1. まず『載せる予定の重さ（積載重量）』を見積もる
    本棚5段＋本＋電装一式が何kgになるかを足し算で出す。本1冊は0.3〜0.5kg。1段に何冊載るかは『棚1段の横幅 ÷ 本1冊の厚み（≒2〜3cm）』で概算（例：幅60cm÷3cm≒20冊）。空欄テンプレを埋める→ 本（◯冊×0.4kg）= ___kg ＋ 棚本体 = ___kg ＋ 電装（Pi・バッテリ・モータ・配線）= ___kg ＝ 合計 ___kg。この合計を、後のステップ3でキットの耐荷重表記と比べる（合計の1.3〜1.5倍以上の耐荷重があるキットを選ぶ＝安全率込み）。
  2. 台車のサイズと重心のイメージを決める
    本棚を載せたとき背が高く重心が上がるので、台車はある程度の幅・奥行きが要る（細長いと倒れやすい）。同時に研究室の通路・ドア幅・曲がり角を実測し、『通れる最大サイズ』も把握しておく。任意でFusion（3D CAD）でざっくり配置を描くと、後段の本棚機構担当と寸法をすり合わせやすい。
  3. 部材・キットを選定する（モータ込みセットを第一候補に）
    ゼロから自作よりまずメカナムホイール＋フレーム＋モータが一式そろったキットを探すのが初心者には堅実。選ぶときは①耐荷重がステップ1合計の1.3〜1.5倍以上 ②台車サイズがステップ2の通れる範囲内 ③モータが付属（または推奨モータ規格が明記） の3点を必ず確認。モータ込みセットなら軸径不一致の手戻りリスクが最初から消える（強く推奨）。
  4. モータとの互換性を『発注前に』確定する（発注前チェックポイント）
    モータを別Issue（駆動・走行制御の担当）が選ぶ場合は、台車を発注する前に『モータの軸径（mm）・取付穴ピッチ』を相互に合意しておく。先に台車だけ発注すると、後決めのモータと軸が合わず作り直しになる。理想はステップ3のモータ込みセットで、この依存自体を消すこと。CONTRIBUTING.mdの『1 PR=1 Issue が目安』に沿い、この合意内容はIssueコメント等に残してPM承認の材料にする。
  5. 見積りをまとめてPMに発注承認をもらう
    候補キット・価格・納期・耐荷重・モータ互換の確認結果を1枚にまとめてPM（水谷）に出し、承認後に発注する。海外発送だと数週間かかることがあるので納期を必ず確認。承認前に勝手に発注しない。
  6. 納期待ちの間にできることを進める
    発注して終わりにせず、待ち時間に: ①重量見積りの精緻化（実物の本・棚で測り直す）②通路・ドア・曲がり角の実測 ③Fusionでの配置確認（任意）④組立工具の事前調達（後述のサイズ確認込み）⑤キットの説明書PDFを先読み・必要なら翻訳。これで届いた瞬間に組み始められる。
  7. 工具をそろえてから組立に着手する
    組立当日に工具不足で止まる典型を避ける。キット付属ネジのサイズを説明書で確認し、対応する六角レンチ・プラスドライバ・スパナをそろえる（多くは2〜5mmの六角レンチセットがあると安心）。そろってからフレーム→ホイール取付の順に組む。
  8. ホイールの向きを配置図どおりに付ける（最重要）
    よくある配置の一例は上から見て各ホイールの外周ローラーの軸が車体中央へ向かい、4輪全体でローラー軸がX字を描く向き（左前=右後ろ、右前=左後ろでローラー軸の向きが揃う＝X配置）。ただしキットによってX配置／O配置のどちらが既定かが異なるので、X字が唯一の正解とは限らない。各ホイールには左用/右用があり、部品の刻印（L/R）と説明書の記号（LF=左前, RF=右前, LR=左後ろ, RR=右後ろ）を1つずつ突き合わせて付ける。『X』か『◇(O)』かで迷わない——最終判断は必ずキット付属の配置図とL/R刻印で行う。これに従う。
  9. 荷重テスト・ガタつき確認をする
    ステップ1で見積もった重さ相当を載せ、たわみ・ガタつき・倒れにくさを『動作確認のしかた』の手順で確かめる。緩みがあれば増し締めする。

📚 進めながら調べる・学ぶこと

  • メカナムホイールの仕組み（なぜ全方向に動けるのか）と受動ローラーの性質
    車輪の外周に斜めのローラーが付いていて、4輪を別々の向き・速度で『動力で』回すと合成された力で横・斜め・その場回転ができる。逆に電源OFFの手押しでは横スライドやその場回転はきれいに出ない（ローラーは受動で自由回転するだけで、純粋な横移動には各輪への駆動入力が要る）。これを先に理解しておくと、組立後の手押しテストで横に動かなくても『組み間違い』と誤判定せずに済む。
    『メカナムホイール 仕組み』『mecanum wheel how it works』で画像付き解説を検索。受動ローラーが自由回転する図と、4輪の駆動方向の組み合わせで進行方向が決まる図を確認する。
  • 必要な積載重量の見積り（本棚5段＋本＋電装）
    耐荷重が足りないキットを選ぶと台車がたわむ・壊れる。逆に過剰だと重く高価。合計重量を出してから1.3〜1.5倍の耐荷重キットを選ぶ、という橋渡しのため。
    本の重さは実物を数冊キッチンスケールで測って平均を取る。電装の品目は README.md / robot/README.md（および本書の用語ミニ辞典『電装』）に載っている部品名を起点にする（Raspberry Pi 5・バッテリ・モータ・モータドライバ・配線・NeoPixel など）。バッテリ・モータドライバ等の細目はそれらに全ては載っていないので、関連する駆動Issue（hw-motor）・本棚機構Issue（hw-shelf-led）で決まる部品リストと各製品ページの仕様（質量 g）を合算する。
  • 筐体サイズと重心（倒れにくさ）
    本棚を載せると背が高く重心が上がり倒れやすい。台車の幅・奥行きと、載せる本棚の高さ・重心位置の関係を理解しないと走行中に転倒する。通路・ドア・曲がり角を通れるサイズ上限の把握も必要。
    『重心 安定 転倒 base of support』で基本（支持基底面が広く重心が低いほど安定）を確認。通路寸法は実測（メジャー）。任意でFusionで配置を描く（製品ページ https://www.autodesk.com/products/fusion-360/ 。使い方は同サイトの学習コンテンツ／チュートリアルを参照）。
  • メカナムホイールの標準配置（X配置/O配置）と左右輪の向き合わせ
    誤った向きで組むと全方向移動が成立しない。『左前=右後ろ／右前=左後ろ でローラー軸を揃える』標準配置を、実際に採用するキットの組立説明書の配置図で裏取りする必要がある（キットにより既定が異なる場合がある）。
    まず『mecanum wheel configuration A/B X/O』で標準配置の図を確認。最終的な正解は購入キット付属の組立説明書（配置図）と部品のL/R刻印。両者を必ず突き合わせる。
  • 部材・キットの選定と発注先
    耐荷重・サイズ・モータ互換・価格・納期を比較して最適なキットを選ぶため。海外発送は納期が数週間かかることがある。
    ロボット台車キットの通販（国内代理店・Amazon・AliExpress・スイッチサイエンス・秋月電子等）で『メカナム シャーシ キット』『mecanum chassis kit』を比較。各商品の耐荷重・寸法・付属モータ規格・納期を表で比較する。
  • 組立手順と必要な工具（サイズまで）
    工具サイズが合わないと組立当日に止まる。事前にそろえるため。
    購入キットの組立説明書PDFで使用ネジのサイズを確認し、対応する六角レンチ・ドライバ・スパナのサイズを特定。一般には2〜5mmの六角レンチセット＋プラスドライバ＋スパナが基本。説明書が外国語なら翻訳して先読みする。
  • モータ取付の互換性（軸径・取付穴ピッチ）
    台車を発注してからモータが合わないと作り直し。発注前に駆動Issue担当と軸径・穴ピッチを合意するため。
    駆動・走行制御Issue担当と直接すり合わせ。モータ込みセットなら付属モータの仕様で自動的に合う。別買いなら、ホイール側のハブ（カップリング）対応軸径とモータの軸径（mm）・出力軸形状（D字/丸軸）を各仕様書で照合する。

つまずきポイントと対処

安全注意（物理作業）

工具での組立はケガに注意。重い荷物（本＋棚）を載せたままの持ち上げ・移動は2人以上で。バッテリやモータの配線は駆動Issueでの作業——この台車Issue段階では通電しない。指をホイールやローラーの隙間に挟まないよう、回転確認は軽く指で。

症状・勘違い原因・対処
手で横に押しても横にスライドしない＝組み間違い？勘違い。電源OFFの手押しでは横スライド・その場回転は出ない（ローラーは受動・自由回転だけ）。横移動の確認はモータをつなぐ駆動Issueで行う。この段階は『4輪が付く・ローラーが自由に回る・たわまない・倒れない』だけを見る。
ホイールの向きが分からない／『X』か『◇(O)』か迷う二択で迷わない。最終判断はキット付属の配置図とL/R刻印で行う。部品のL/R刻印と説明書の記号（LF/RF/LR/RR）を1輪ずつ突き合わせる。よくある一例は上から見てローラー軸がX字を描くX配置（左前=右後ろ、右前=左後ろが同じ向き）だが、O配置を既定とするキットもあるので、X字＝唯一の正解とは限らない。
台車を発注したらモータの軸が合わなかった発注前にモータの軸径・穴ピッチを駆動Issue担当と合意していない。発注前チェックポイントを守る。最初からモータ込みセットを選べばこのリスクは消える。
載せたらフレームがたわむ・きしむ耐荷重不足。ステップ1の合計重量の1.3〜1.5倍以上の耐荷重キットを選び直す。ネジの締め忘れ・緩みも増し締めで確認。
背が高くてグラグラ・倒れそう重心が高い／台車の幅が狭い。台車サイズを見直す、重い電装を下段に置いて重心を下げる。走行安定は駆動Issueで再評価。
組立当日に工具が足りず止まるネジサイズを説明書で先に確認していない。ステップ7で六角レンチ（2〜5mmセット）等を事前調達してから着手。
発注したのに数週間届かない海外発送の納期未確認。ステップ5で納期を必ず確認。ステップ6の『納期待ちにできること』で待ち時間を空転させない。

動作確認のしかた

この段階は動力なしでの完成判定。横移動・その場回転は別Issue（駆動）で確認するので、ここでは見ない。

  1. 4輪がフレームに固定され、各ホイールの外周ローラーを指で回すと4輪とも斜め方向に自由に回る（引っかからない）ことを確認する。
  2. 上から見て、ローラー軸の向きが配置図どおり（対角で揃う／全体でX字）になっているか、説明書の図と1輪ずつ照合する。
  3. 前にまっすぐ押すと真っ直ぐ転がることを確認する（前後の直進は手押しでも出る）。
  4. ステップ1で見積もった重さ相当の荷物を載せ、フレームの目視たわみが数mm以内で、外すと元に戻る（永久変形しない）ことを確認する。
  5. 荷物を載せたまま台車を手で軽く揺すり、ガタつき音・ぐらつきが無いことを確認する（あれば増し締め）。
  6. 荷物を載せて静止させ手を離し、自然に倒れない（重心が安定）ことを確認する。
ここまでできたらDone

上の6つが満たせれば、丈夫でガタつかない全方向移動用の台車が物理的に完成。次は駆動Issueでモータをつないで実際に全方向に動かす。

用語ミニ辞典

言葉かんたんな意味
メカナムホイール外周に斜めのローラーが付いた車輪。4輪を別々に『動力で』回すと前後・左右・斜め・その場回転ができる（横移動には各輪の駆動入力が必要）。
ローラー（外周ローラー）メカナム車輪の外側に並ぶ小さな樽型の部品。自由回転する（受動）。この軸の傾きの向きが組立の肝。
X配置 / O配置メカナム4輪のローラー軸の向きの組み方。上から見てローラー軸がX字を描くのがX配置（左前=右後ろ、右前=左後ろが同じ向き）、逆向きがO配置。どちらを既定とするかはキットにより異なるので、X配置＝唯一の標準とは限らない。最終確認は必ずキット付属の配置図とL/R刻印で。
耐荷重そのキット・台車が安全に載せられる重さの上限。見積り合計の1.3〜1.5倍以上を目安に選ぶ。
重心 / 支持基底面重さの中心 / 接地している脚の囲む範囲。重心が低く支持基底面が広いほど倒れにくい。
軸径 / 取付穴ピッチモータの回転軸の太さ(mm) / ネジ穴の間隔。台車・ホイールと合わないと取り付かない。発注前に合意する。
機構機械の動く仕組み・骨組みの設計（フレームや本棚の載せ方など物理構造のこと）。
電装電気部品まわり一式：Raspberry Pi 5・バッテリ・モータ・モータドライバ・配線・NeoPixel(LED) など。
モータドライバ弱い制御信号でモータの強い電流を制御する中継部品（駆動Issueで使う）。
Fusion（Fusion 360）Autodeskの3D CADソフト。台車や配置を立体で設計できる（このIssueでは任意）。
