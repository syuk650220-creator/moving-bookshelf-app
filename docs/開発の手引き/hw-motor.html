このIssueは何？

ゴールは、ロボの足回り（走行用モータ）を選んで配線し、Raspberry Pi 5 から出した信号で各モータを「向き（前進／後退）」「速度（速い／遅い）」ともに回せる状態にすること。ロボが自分で動くための一番土台の部分で、ここが動かないと自律走行(Nav2)も呼び出しも始まらない。

やることは大きく2つ。(1)選定: 車重・車輪径から必要なトルク／回転数をざっくり見積もり、それに合うDCモータ・モータドライバ・電源を選ぶ。(2)配線: モータ→ドライバ→電源→Pi5(制御信号) をつなぎ、テスト信号で回るか確かめる。物理作業＋電気作業の両方が混ざるIssue。

足回りはメカナム4輪（台数・ch数・電源容量に直結）

本ロボの足回りは hw-chassis でメカナムホイール4輪・全方向移動に確定している。メカナムは4輪を個別に正逆＋PWM速度制御して初めて横移動・斜行・その場回転が成立する設計なので、選定では『最低4系統のドライバch（または2chドライバ×複数）』『ストール電流×4輪を見込んだ電源容量』が要る。1〜2個前提で部品を選ぶと後で台数・ch数・容量が足りず手戻りになる。本Issueの単体テストは1輪ずつ回せれば可。全方向移動の合成制御（4輪の協調）は後続のロボ制御ソフト/Nav2 Issueで行うので、本Issueでは『各輪を個別に向き・速度制御できる』までを担当する。

このIssueの心臓部＝「テスト信号を自分で出せること」

Doneも動作確認も『Pi5から出したテスト信号でモータが回る』に依存している。本Issueでは、Raspberry Pi 5 から数行のPythonスクリプトでGPIOピンをHIGH/LOWして向きを決め、PWM(後述)でデューティ比を変えて速度を変える、という最小のやり方を主軸にする。どのライブラリのどのページを読むかは『調べる・学ぶこと』で道案内する（答えのコードは載せない）。

Pi5はGPIO/PWMの仕組みが従来機と別物（最重要の罠）

本プロジェクトは Raspberry Pi 5 に確定。前提として、このPiは hw-pi5-setup で Ubuntu Server＋ROS2 構成（ヘッドレス）にしてあるので、Raspberry Pi OS 専用ツールはそのままでは挙動が違う点に注意（本Issueの単体テストは ROS2 を使わず Ubuntu 上の Python から GPIO/PWM を出してよい）。Pi5 はI/Oを新しい RP1 チップが担当するため、Pi4以前で定番だった RPi.GPIO がそのままでは動かないことがあり、ハードウェアPWMの出し方も変わっている。ネットの古いPi4向け記事を真似しても動かず延々ハマるのが典型（アプリ側の『このNext.jsは最新版で君の知る書き方と違う』警告とまったく同じ構図）。ライブラリは Ubuntu×Pi5(RP1) で動くもの＝gpiozero+lgpio を第一候補にする。pigpio は Pi5(RP1) で動作しないことがあるため、選ぶ前に必ずPi5対応状況を確認すること。

完了の定義（Done）

  • 車重・車輪径・目標速度から、必要トルク／回転数をざっくり見積もった（足回りはメカナム4輪前提で、台数＝4を見込んだ。数値の根拠を1枚メモに残す）
  • 見積りに合うDCモータ・モータドライバ・電源を選び、選定理由（電流定格・対応電圧・PWM対応、4輪ぶんのドライバch数とストール電流×4輪の電源容量）をメモした
  • ドライバの制御入力(IN)が 3.3Vロジックで駆動できることを確認した（Pi5は3.3V。5V前提のドライバならレベル変換が要る）
  • Pi・モータドライバ・モータ電源のGNDを共通接続した（共通GND）——これを忘れると信号が通らない最頻出ミス
  • Pi5から出したテスト信号で、各モータが 前進／後退 できた（向き制御）
  • 各モータの速度を PWMで変えられた（低速／高速の最低2段。ハードウェアPWMが使えればそれを、難しければ gpiozero のソフトウェアPWM(PWMOutputDevice 等)でも可。できればテスターでIN/PWMピンの電圧変化も確認）
  • 通電前にテスターで配線チェック（VM電源が来ているか・IN/方向ピンに電圧が出るか・GND導通）をした
  • 逆起電力対策・ヒューズなど安全配線の方針を決め、反映した（『調べる・学ぶこと』参照）
単体で満たせる

ここでの『テスト信号』はロボ制御ソフト(Nav2等)が無くても、Pi5から手書きの数行スクリプトで出せばよい。後続のロボ制御ソフトIssueが未完成でも、このIssue単体で Done を満たして検証できる。

進め方ステップ

  1. 前提（仮値）をPM・ハ設で先に合意する
    前提Issue: hw-pi5-setup（Piが起動しSSHで操作できる状態。order 1 / Sprint 0）が先に終わっていること。物理的に直結する hw-chassis（メカナム筐体）とは、モータ軸径・取付穴ピッチを発注前に相互合意する。順序の目安は『pi5-setup→（chassisと並行で）motorの選定→配線』。見積りの入力になる『車重(おおよそ)・車輪径・目標速度・段差/坂の有無』を先に決める。本ロボは屋内・平床の図書館移動ロボなので、段差を越えないなら登坂角は0でよい（坂や段差を越えるなら必要トルクが跳ね上がるので別途見積り）。目標速度は『歩行速度の半分=0.5m/s程度』あたりを出発点に仮置きして、走らせながら見直す前提でOK。
  2. 必要トルク／回転数をざっくり見積もる
    車重と車輪径から、車輪を回すのに要る力→トルクを計算する。考え方の型は『駆動力を出す→車輪半径を掛けてトルクにする→安全率（余裕）を掛ける』。回転数は『目標速度 ÷ 車輪円周』から必要なrpmを出す。式と手順は research の『必要トルク・回転数の見積り』を参照。数字は仮でよいが、根拠をメモに残す。
  3. モータの種類を決める
    DCギヤードモータ（減速機付き）か、エンコーダ付きか等を決める。自律走行(Nav2)では速度の実測（オドメトリ）にエンコーダが効くので、後続を見据えてエンコーダ付きを検討する価値がある。research の『モータの種類』参照。
  4. モータドライバを選ぶ（電流が肝）
    ドライバが流せる電流が、モータのストール電流（＝モータが回れず止まった時に流れる最大電流。ここが選定の上限基準）に耐えられるかを最優先で確認する。あわせて『対応電圧』『PWM入力に対応するか』『制御入力(IN)が3.3Vロジックで動くか（Pi5は3.3V／5V前提ならレベル変換が要る）』を確認する。research の『モータドライバの選定』参照。
  5. 電源を設計する（電圧・容量・分配・初回テスト電源）
    モータ用電源（VM）と Pi5用電源（5V系）は分けて考える。容量はモータのストール電流×台数＝メカナム4輪を基準に見込む（4輪が同時に大電流を引く前提）。初回の単体テストは、可能なら電流制限付きの安定化（ベンチ）電源から始める——いきなり大容量バッテリ直結＋大電流は、配線ミス時のダメージが大きい。research の『電源設計』参照。
  6. 配線図を描く（共通GNDを明記）
    モータ→ドライバ→電源→Pi5(制御信号) の接続を、配線図ツールで描く。この図に『電源GNDとPi5のGNDを必ず共通にする（共通GND）』を必ず描き込む——これを忘れると信号が通らない最頻出ミス。逆起電力対策(フライバックダイオード等)・ヒューズの位置もここで決める。
  7. 仮配線して『最小のテスト信号』で単体テスト
    ブレッドボード等で仮配線し、Raspberry Pi 5 から数行のPythonスクリプトを書いて、(a)方向ピンをHIGH/LOWして向きを決める (b)PWMピンのデューティ比を変えて速度を変える という最小テストをする。どのライブラリ(gpiozero/lgpio/pigpio 等)でPi5のGPIO/PWMを出すかは research の『Pi5でGPIO/PWMを出す手段』で確認（答えコードは書かない）。通電前に必ずステップ8のチェックを先にやる。
  8. 通電前チェック→本配線へ
    通電前にテスターでショート確認・GND導通・電圧の向きを点検。問題なければ本体への本配線に進む（端子の締結・はんだ・コネクタ化）。本配線後も共通GND・ヒューズ・逆起電力対策が反映されているか再確認する。

📚 進めながら調べる・学ぶこと

  • Raspberry Pi 5（Ubuntu）でGPIO/PWMの制御信号を実際に出す手段（本Issueの動作確認の根幹）
    テスト信号＝Pi5からGPIOをHIGH/LOWして向きを決め、PWMでデューティ比を変えて速度を変える、が本Issueの心臓部。これが出せないとDoneも動作確認も始まらない。前提のPiは hw-pi5-setup で Ubuntu＋ROS2 構成なので、Ubuntu×Pi5(RP1) で動くライブラリをまず決める必要がある。
    第一候補は gpiozero（公式: https://gpiozero.readthedocs.io/ ）＋下回り lgpio（Ubuntu×Pi5 で素直に動く組合せ）。pigpio は Pi5(RP1) で動作しないことがあるので同列候補にはせず、使うなら対応状況を先に確認すること。Pi5でハードウェアPWMを使うなら対応ピン・対応状況を必ず確認（取れない場合は gpiozero のソフトウェアPWM=PWMOutputDevice 等で代替可）。Pi5公式: https://www.raspberrypi.com/documentation/ の『Raspberry Pi hardware → GPIO』節。
  • Raspberry Pi 5 のGPIO/PWMが従来機(Pi4以前)と違う点（RP1の罠）
    Pi5は I/O を RP1 チップが担当し、定番だった RPi.GPIO がそのまま使えないことがある。古いPi4記事・RPi.GPIO前提のサンプルを真似して『動かない』で延々ハマるのを避けるため、最初に違いを把握する。
    Pi5公式 https://www.raspberrypi.com/documentation/ の『Raspberry Pi hardware → GPIO』『General → 各モデルの違い』。検索する時も必ず『Raspberry Pi 5』を付け、Pi4/Pi3向け記事を鵜呑みにしない。
  • Piの3.3Vロジックとモータドライバ制御入力(IN)の電圧整合（レベル変換の要否）
    Pi5のGPIOは3.3V出力。ドライバのIN側が5Vロジック前提だと信号がしきい値に届かず回らない／不安定になる。選定段階で気づかないと配線後に詰まる。レベル変換(レベルシフタ)が要るかを独立に判断する必要がある。
    候補ドライバのデータシートの『Logic input voltage / VIH(High入力電圧)』を読む。Pi5のGPIO電圧は https://www.raspberrypi.com/documentation/ のGPIO節（3.3V）で確認。3.3VでVIHを満たさないならレベル変換を検討。
  • 必要トルク・回転数の見積り（屋内平床前提）
    モータ選定の入力。過小だと登れない/動かない、過大だと電源とコストが無駄。前提（車重・車輪径・目標速度・段差有無）を決めてから計算する。屋内平床で段差を越えないなら登坂角は0として加速・転がり抵抗中心で見積もる。
    計算の型: ①必要な駆動力（車重×加速・転がり抵抗）→②駆動力×車輪半径＝必要トルク→③安全率を掛ける、の順。回転数は目標速度÷車輪円周→rpm。検索語『DCモータ トルク 計算 車輪径 駆動力』に加え、用語を『torque / rpm / wheel radius』で英語ページも当たると式が見つかりやすい。
  • モータの種類（DCギヤード／エンコーダ付き）
    減速機付き(ギヤード)でトルクを稼ぐのが定番。さらにエンコーダ付きなら速度の実測ができ、後続の自律走行(Nav2)のオドメトリ（自分がどれだけ進んだかの推定）に効く。先を見据えて選ぶ。
    Nav2でオドメトリが要る背景: https://docs.nav2.org/ 。エンコーダの役割は『rotary encoder odometry wheel』で調べる。
  • モータドライバの選定（電流定格・PWM）
    ドライバが流せる電流がモータのストール電流に耐えないと焼ける/保護がかかって止まる。PWMで速度制御できるか、対応電圧（VM）が電源に合うかも必須条件。
    Hブリッジ(正逆転回路)とPWM入力の考え方を学ぶ。具体的な型番はストール電流から自分で候補を絞るのが本筋。学習用に1〜2例のデータシートを読み、『連続/ピーク電流・対応電圧・PWMピンの使い方・IN側ロジック電圧』の読み方を身につける程度に留める。
  • 電源設計（電圧・容量・分配・初回テスト電源）
    電圧が合わないと動かない/壊れる、容量不足だと電圧降下でリセットや脱調。モータ用とPi用を分けて考える。初回テストの電源選びは安全に直結する。
    Pi5の給電: https://www.raspberrypi.com/documentation/ の『Raspberry Pi hardware → Power supply』（純正は5V/5A(27W)推奨。5A未満だとUSB周辺機器の電流が制限される、という条件を各自で確認）。初回テストは電流制限付き安定化電源→問題なければバッテリ、の順が安全。
  • 配線の安全（逆起電力・フライバックダイオード・ヒューズ）
    モータは止まる瞬間に逆向きの電圧(逆起電力)を出し、ドライバを壊しうる。フライバックダイオードやドライバ内蔵保護で逃がす。過電流に備えヒューズを入れる。事故と部品破損を防ぐ核心。
    『back EMF flyback diode motor driver』『DC motor fuse sizing』で原理を確認。多くのドライバICは保護を内蔵するのでデータシートの『protection / freewheeling diode』節も読む。
  • テスター(マルチメータ)での基本的な計測
    回らない時の原因切り分けに必須。VM電源が来ているか、IN/方向ピンに電圧が出ているか、GNDが導通しているかを測れると、自分で詰まりを解ける。
    『マルチメータ 使い方 直流電圧 導通』『multimeter DC voltage continuity test basics』で、電圧測定(並列に当てる)と導通チェックの基本を確認。

つまずきポイントと対処

症状原因・対処
モータが回らない／弱い最頻出は共通GND忘れ。Pi5・ドライバ・モータ電源のGNDを必ず1点で共通にする。次に多いのが信号が3.3VでドライバのIN側しきい値に届かない（レベル変換が要るケース）。VM電源が来ているかもテスターで確認。
古いPi4記事どおりに書いたのに動かないPi5はRP1採用でGPIO/PWMの仕組みが従来機と別物。RPi.GPIOがそのまま動かないことがある。Pi5対応ライブラリ(gpiozero+lgpio等)とPi5公式ドキュメントを見る。検索語に必ず『Raspberry Pi 5』を付ける。
向きは変わるが速度が変わらないPWMピンに本当にPWMが出ているかをテスターやロジックで確認。ハードウェアPWM対応ピン/ライブラリの組合せがPi5で正しいか確認（research『Pi5でGPIO/PWM』）。
ドライバが熱い／保護で止まるストール電流（モータが止まった時の最大電流）がドライバ定格を超えている可能性。連続/ピーク電流定格を見直す。放熱・配線太さも確認。
通電した瞬間に煙・部品破損配線ミス（極性逆・ショート・GND未共通）。通電前のテスターチェックを必ず先に。逆起電力対策(フライバック/内蔵保護)とヒューズが入っているか確認。
バッテリ直結でテストして壊した初回テストは電流制限付きの安定化電源から。いきなり大容量バッテリ直結＋大電流は事故時のダメージが大きい。小容量電源＋ヒューズから始める。
安全上の注意（必ず守る）

①通電前に必ずテスターでショート・極性・GND導通を確認。②初回テストは電流制限付き安定化電源（無ければ小容量電源＋ヒューズ）から。③配線をいじる時は必ず電源を切る。④モータは急に動く——回転部・配線に指や髪を巻き込まれないよう固定してから通電する。⑤ヒューズと逆起電力対策を省略しない。

動作確認のしかた

完成判定は『Pi5のテスト信号で各モータが向き・速度ともに制御できる』こと。下の手順で、通電前チェック→単体テストの順に進める。

  1. 【通電前・テスター】VM電源ラインに正しい電圧が来るか、IN/方向ピンに（信号を出した時）電圧が出るか、Pi・ドライバ・電源のGNDが導通しているかを測る。ショート・極性逆がないことを確認。
  2. 【初回電源】可能なら電流制限付きの安定化電源から給電する（無ければ小容量電源＋ヒューズ）。電流計があれば消費電流を見ながら徐々に上げる。
  3. 【向き】Pi5から方向ピンをHIGH/LOWに切り替えるテスト信号を出し、各モータが前進／後退すること（左右・全モータ分）を目視確認。
  4. 【速度】PWMのデューティ比を低/高の2段に変え、回転が遅い/速いと変わること（触って軽く温かい程度で異常発熱がない）を確認。可能ならテスターでPWM/INピンの電圧変化も見る。
  5. 【安全】テスト中に異音・異常発熱・煙が無いこと。ヒューズが飛ばず、逆起電力対策が効いていること。
  6. 【記録】使った電源電圧・電流、選定したモータ/ドライバ型番、配線図を1枚にまとめて後続Issue（ロボ制御ソフト）へ引き継ぐ。

🤖 AIコーチの使い方

用語ミニ辞典

言葉かんたんな意味
トルク回そうとする力の強さ（N·m）。重い車体・大きい車輪ほど多く要る。
ストール電流モータが回れず止まった(拘束された)時に流れる最大電流。ドライバ/電源選定の上限基準になる。
回転数(rpm)1分あたりの回転数。目標速度÷車輪円周から必要rpmが出る。
ギヤード（減速機付き）モータ歯車で回転を遅くする代わりにトルクを増やしたモータ。走行用の定番。
エンコーダ回転量を測るセンサ。速度の実測＝自律走行のオドメトリに使える。
オドメトリ自分がどれだけ進んだか（位置・速度）の推定。エンコーダで車輪の回転量を実測して求める。自律走行(Nav2)で効く。
モータドライバ／HブリッジPiの弱い信号で大電流のモータを動かす中継回路。Hブリッジで正逆転(向き)を切り替える。
PWM／デューティ比ONとOFFを高速で切り替えて平均の力を変える方式。ON時間の割合(デューティ比)を上げると速くなる。
VM / ロジック電圧VM=モータを回す電源電圧。ロジック電圧=ドライバの制御入力(IN)が反応する電圧。Pi5は3.3V。
VIHドライバが入力を「High」と判定する最低入力電圧。Pi5の3.3VがこのVIHを上回らないと信号をHighと認識できず、レベル変換が要るかどうかの判定値になる（データシートで見る数値）。
共通GNDPi・ドライバ・電源のGND(0V基準)を1点でつなぐこと。これが無いと信号が通らない。
逆起電力モータが止まる瞬間にコイルが出す逆向きの電圧。放置するとドライバを壊しうる。
フライバックダイオード逆起電力を安全な経路へ逃がして部品を守る保護ダイオード（多くのドライバICは同等の保護を内蔵）。
RP1Pi5のI/Oを担当する新チップ。これによりGPIO/PWMの扱いがPi4以前と変わった。
レベル変換(レベルシフタ)3.3Vと5Vなど電圧の違う信号同士をつなぐための変換回路。
