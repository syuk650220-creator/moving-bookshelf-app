このIssueは何？

ゴールは、移動本棚ロボの電子回路を確定して基板の製造データまで作ること。流れは2段階。Phase1: まずブレッドボード（はんだ付けせず挿すだけの実験ボード）で実際に配線し、各ブロック（Pi5のGPIO・モータドライバ・NeoPixel・電源）が想定どおり動くか通電して確かめる。Phase2: 配線が確定したら KiCad で回路図→基板レイアウト→DRC（設計ルールチェック）を通し、製造データ（ガーバー）を出力する。

なぜいきなり基板化しないか。基板は一度作ると配線を直せない。先にブレッドボードで『この配線で本当に光る/回る』を確認してから基板に焼き付ける、というのが事故と作り直しを防ぐ鉄則。配線図は必ず残す（後でKiCadに写すため・引き継ぎのため）。

最重要・最初に決める：NeoPixelをPi5でどう光らせるか

Raspberry Pi 5 は新しい『RP1』チップでGPIOの仕組みが変わり、旧Pi向けの定番ライブラリ rpi_ws281x の通常PWM/DMA方式がそのままでは動かない。よく貼られる Adafruit『NeoPixels on Raspberry Pi』は旧Pi前提なので、その通り配線・検証しても『データシート通りなのに光らない』で必ず詰まる。Pi5では(1)RP1対応のカーネルモジュール版 rpi_ws281x (2)PIO駆動ドライバ (3)SPI接続(GPIO10)で動く Pi5Neo 等、別の手段が要る。どの方式を使うかでデータピンが変わる（例: SPIなら GPIO10）。基板にピンを焼き付けるIssueなので、ここを決めずに配線すると基板を作り直す。LED制御(NeoPixel)担当と早めにピン番号をすり合わせ、駆動方式の確定を最初の前提タスクにする。

前提（先に確定すべきこと）

回路設計は『部品が決まっていること』が前提。(1)モーター・ドライバの選定 (2)NeoPixelの駆動方式とデータピン (3)『NeoPixel×5』が5ピクセルなのか5本のテープ（各何ピクセル）なのか（電流が桁で変わる） (4)電源の種類。ロボは自走する＝電池駆動なので、据置ACではなく可搬電源（モバイルバッテリ/Li-ion等）前提で電圧・容量(mAh)・連続放電電流を見積もる。これらが固まらないと配線図も電源容量も決められない。

完了の定義（Done）

このIssueのゴール範囲: KiCadでDRCを通した製造データ（ガーバー）出力まで。実基板の発注・はんだ実装・通電確認は別Issue（またはPM判断）とする。発注はブレッドボード検証が終わってから、というプロジェクトのルールに合わせ、Sprint2ではここまでで完結させる。

  • 【前提確定】モータドライバ選定済み／NeoPixel駆動方式とデータピン確定済み／『NeoPixel×5』の単位（ピクセル数）確定済み／電源（電池）の電圧・容量決定済み
  • 【Phase1】ブレッドボードで主要部品（Pi5該当GPIO・モータドライバ・NeoPixel・電源・レベルシフタ・コンデンサ/抵抗）を結線し通電できている
  • 【Phase1】各ブロックが想定どおり動く（モーターが正転・逆転する／NeoPixelが指定の色で光る）。確認用の最小テストコードはLED制御・モータ制御担当と連携して用意
  • 【Phase1】配線図を残した（手書き写真でも可。できればFritzingで清書）
  • 【Phase2】KiCadで回路図(schematic)を起こした
  • 【Phase2】各部品にフットプリントを割り当て、PCBレイアウトを作成した
  • 【Phase2】DRC（設計ルールチェック）が通り、未配線(ratsnest)が無い
  • 【Phase2】製造データ（ガーバー＋ドリル）を出力した（＝このIssueのゴール）

進め方ステップ

  1. 【前提】NeoPixel駆動方式と部品を確定する
    まず研究項目『Pi5でのNeoPixel駆動』を読み、(1)RP1対応版rpi_ws281x (2)PIO駆動 (3)SPI接続のPi5Neo のどれを使うかをLED制御担当と決める。これでデータピン（SPIならGPIO10等）が決まる。あわせてモータドライバ、電源（電池）、『NeoPixel×5』の単位（ピクセル数）も確定。ここが決まらないと先に進まない。
  2. 使う部品とデータシートを集める
    Pi5・モータドライバ・モーター・NeoPixel・レベルシフタIC(74AHCT125等)・コンデンサ/抵抗をリストにし、各データシートを集める。データシートからピン配置・動作電圧・必要電流を読む（研究項目参照）。受動部品（NeoPixel電源のデカップリングコンデンサ・データ線の直列抵抗）も忘れずリストに入れる。
  3. 電源系統と電流を紙に整理する
    『どこから何Vが出て、どこに行くか』を1枚の紙に描く。Pi5・モーター・NeoPixelは消費電流が違う。NeoPixelは1ピクセルあたり最大約60mA（フルカラー白・最大輝度）。必要電流≒『LED総数×60mA ＋ モーター分 ＋ Pi5分（約3A目安）』で概算する。テープなら本数×ピクセル数で数A級になることも。ロボは電池駆動なので、この合計に耐える電圧・容量(mAh)・連続放電電流の電池を選ぶ。USB給電では足りない前提。
  4. ブレッドボードにブロックごとに配線する（一気に全部やらない）
    まず電源とGNDの共通化、次にPi5↔モータドライバ、最後にNeoPixel、とブロック単位で配線する。3.3V→5VのNeoPixelデータ線は基本レベル変換が要る（手順5）。各ブロックの動作確認には簡単なテスト用コードが必要なので、LED制御(NeoPixel)・モータ制御の担当者と連携し、点灯・回転の最小サンプルを先に用意してから通電する。配線中はこまめに写真を撮って配線図の元にする。
  5. レベル変換（3.3V→5V）を入れる
    Pi5の信号は3.3V、NeoPixelは5Vロジック。Pi(3.3V)→NeoPixel(5V)のデータ線は基本的にレベル変換が必要と考えてよい。定番は74AHCT125等のロジックレベルコンバータ（または最初の1個だけ別対応する方法）。要否の目安は『信号線が長い・本数が多い・電源5V』なら入れる。具体的な部品・抵抗値はデータシートとAdafruitの記事で裏取りする（研究項目参照）。
  6. テスターで通電・電圧を確認する
    テスター（マルチメータ＝電圧や導通を測る道具）の使い方：黒い棒を『COM』端子、赤い棒を『VΩ』端子に挿す。配線がつながっているか見るときはダイヤルを導通モード（ブザー記号）に、電圧を測るときは直流電圧 DCV（V⎓ 記号）に合わせる。GND同士の導通、各電源ラインの電圧（3.3V/5V/電池電圧）が想定どおりかを1つずつ確認する。
  7. 各ブロックを通電して動作確認する
    用意した最小テストコードで、モーターが正転・逆転するか、NeoPixelが指定の色で光るかを確認する。1ブロックずつ。動かなければ手順6のテスターで電圧・導通から切り分ける。全ブロックOKになったら配線が『確定』。
  8. 配線図を清書する（Phase1完了）
    確定した配線を残す。手書き写真でも最低限OKだが、できればFritzingでブレッドボード図/回路図に清書すると次のKiCad化が楽。これでPhase1のDoneを満たす。
  9. 【Phase2】KiCadで回路図(schematic)を起こす
    確定した配線をKiCadのスキマティックエディタで部品を置いて結線する。チームで使うKiCadのバージョンを揃える（8系/9系でUIや手順が変わり、チュートリアルと食い違って混乱しやすい）。ライブラリに無い部品（モータドライバIC等）はシンボル/フットプリントを入手or自作する（研究項目参照）。
  10. 【Phase2】フットプリント割当→PCBレイアウト→DRC→ガーバー出力
    各部品に実物の大きさ＝フットプリントを割り当て、PCBエディタへ。部品を配置し配線（ratsnestの白線を実配線に）。DRC（設計ルールチェック）を実行してエラー0・未配線0にする。最後に製造データ（ガーバー＋ドリル）を出力したらこのIssueのゴール。

📚 進めながら調べる・学ぶこと

  • 【最重要】NeoPixel(WS281x)をRaspberry Pi 5で光らせる正しい方法
    Pi5はRP1チップでGPIOの仕組みが変わり、旧Pi向けの定番 rpi_ws281x の通常PWM/DMA方式がそのままでは動かない。よく貼られるAdafruit汎用ガイドは旧Pi前提で、その通りやると『データシート通りなのに光らない』で確実に詰まる。どの接続方式（GPIO直/SPI/PIO/外部マイコン）・どのライブラリ対応版を使うかでデータピンが変わるため、配線（=回路設計）を確定する前に駆動方式を決める必要がある。
    rpi_ws281xのPi5対応Wiki https://github.com/jgarff/rpi_ws281x/wiki/Raspberry-Pi-5-Support と issue #528 https://github.com/jgarff/rpi_ws281x/issues/528 。SPI(GPIO10)で動かす Pi5Neo https://github.com/vanshksingh/Pi5Neo 。Adafruitの汎用ガイド https://learn.adafruit.com/neopixels-on-raspberry-pi はPi5前提でない点に注意して読む。
  • 3.3V→5VのNeoPixelデータ線のレベル変換（要否の判断と方法）
    Pi5の信号は3.3V、NeoPixelは5Vロジックなので、データ線は基本的にレベル変換が必要と考えてよい。スキップするとデータが不安定で『たまに光る/色が化ける』になる。基板に何を載せるかに直結する。
    Adafruit NeoPixel Best Practices（Powering / Logic Levelの章）https://learn.adafruit.com/adafruit-neopixel-uberguide/best-practices と レベルシフタ記事 https://learn.adafruit.com/neopixel-levelshifter 。検索語『74AHCT125 logic level shifter NeoPixel』。要否の目安（信号線の長さ・本数・電源電圧）も確認。
  • NeoPixelの電流見積りと電源容量(A数)の計算
    電源（電池）の容量を決めるのに必須。1ピクセルあたり最大約60mA（フルカラー白・最大輝度）。総数×60mA＋モーター分＋Pi5分(約3A目安)で必要電流を概算する。『NeoPixel×5』がピクセル数かテープ本数かで桁が変わるので前提確定が先。USB給電では足りない理由もこの数字でつながる。
    Adafruit NeoPixel uberguide『Powering NeoPixels』https://learn.adafruit.com/adafruit-neopixel-uberguide/powering-neopixels 。具体数値は使う部品のデータシートで裏取り。
  • NeoPixel回路の定番受動部品（コンデンサ・抵抗）
    Adafruitが推奨する定番。電源には大容量コンデンサ（数百〜1000µF）でデカップリング（突入電流対策）、データ線には直列抵抗（約330〜470Ω）で信号保護。これらはデータシートに載らない『小物』なので存在に気づかないと部品リストから漏れる。
    Adafruit NeoPixel Best Practices https://learn.adafruit.com/adafruit-neopixel-uberguide/best-practices の Powering / Adding a Capacitor / Adding a Resistor の節。
  • テスター（マルチメータ）の基本操作：導通チェックと電圧測定
    step6の通電確認に必須。モードの切替（導通ブザー／直流電圧DCV）と棒を挿す端子（黒=COM／赤=VΩ）を知らないと測定そのものができない。
    検索語『マルチメータ 使い方 導通 電圧 初心者』。手持ちのテスターの取扱説明書（DCV/導通モードの記号）も必ず1度読む。
  • データシートでピン配置・動作電圧・必要電流を読む
    配線の正しさと電源容量の根拠になる。各ピンが何V・何mAを想定しているか、絶対最大定格を超えないかを読む基本。
    各部品メーカー配布のデータシート（型番＋datasheetで検索）。Pi5のピン配置は『Raspberry Pi公式ドキュメントのGPIOの章（raspberrypi.com/documentation 内）』を参照（検索語『Raspberry Pi 5 GPIO documentation pinout』）。Pi5のGPIOは3.3Vで5V非耐性＝5Vを直接入れない点に注意。
  • ブレッドボードの定格電流上限とモーター大電流の扱い
    ブレッドボードは大電流に弱く、目安約1A程度。モーターの大電流を直接流すと接点が発熱・溶けて事故や誤判定になる。
    検索語『ブレッドボード 定格電流 限界』。モーター電源は端子台やワニ口で別配線し、ブレッドボードは信号と小電流だけに使うのが安全。
  • KiCadのバージョン差と、ライブラリ未収録部品のシンボル/フットプリント
    KiCadは8系/9系でUI・操作が変わり、古い/新しいチュートリアルと食い違って混乱する典型。またモータドライバIC等はライブラリに無いことが多く、独自部品で必ず詰まる。
    KiCad公式 https://docs.kicad.org/ （使うバージョンのドキュメントを開く・チームでバージョンを揃える）。未収録部品はメーカー配布の KiCad ライブラリや SnapEDA（snapeda.com）等で入手、無ければ自作。検索語『KiCad symbol footprint not in library』。
  • KiCadの全体の流れ（回路図→フットプリント割当→PCB→DRC→ガーバー）
    Phase2のゴールまでの道筋。各ステップの意味（DRC=設計ルール違反を機械チェック、ratsnest=未配線の白線）を理解すると詰まりにくい。
    KiCad公式 https://docs.kicad.org/ のGetting Started / PCB Editor。検索語『KiCad getting started schematic to gerber』。
  • 電池駆動（可搬電源）の設計制約
    ロボは自走するためACではなく電池。電圧・容量(mAh)・連続放電電流・保護回路を見積もらないと、走行中に電圧降下で誤動作する。
    検索語『Li-ion 連続放電電流 C レート 選び方』『モバイルバッテリ 最大出力電流』。リチウム電池の取り扱い注意（後述の安全callout）と合わせて確認。

つまずきポイントと対処

症状・勘違い原因・対処
NeoPixelがデータシート通りなのに光らないPi5の最大の落とし穴。RP1チップで旧来の rpi_ws281x 通常PWM/DMA方式が動かない。Pi5対応版・PIO駆動・SPI(Pi5Neo)のどれかに切替。Adafruit汎用ガイドはPi5前提でない。駆動方式を先に確定する（前提タスク）。
NeoPixelの色が化ける・たまにしか光らない3.3V→5Vのレベル変換をしていない可能性大。74AHCT125等のレベルシフタを入れる。データ線に直列抵抗(約330〜470Ω)が無いのも原因。
LEDが暗い・チラつく／走行中に誤動作電流不足。NeoPixelは1ピクセル最大約60mA。総数×60mA＋モーター＋Pi5(約3A)で見積もり直す。電池の連続放電電流が足りているかも確認。USB給電は足りない。
ブレッドボードが熱い・溶けたブレッドボードは大電流(目安1A)に弱い。モーターの大電流を直接流さない。モーター電源は端子台やワニ口で別配線し、ブレッドボードは信号と小電流だけに使う。
Pi5のGPIOが壊れたPi5のGPIOは3.3Vで5V非耐性。5Vを直接GPIOに入れない。レベル変換やドライバを介す。
『配線はできたが光らせる/回す方法が無い』で止まる回路検証にはソフトでGPIOを叩く必要がある。LED制御・モータ制御担当と連携し、点灯・回転の最小テストコードを先に用意してから通電する。
デカップリングコンデンサ/直列抵抗の存在を知らず部品が足りないデータシートに載らない定番の受動部品。電源に大容量コンデンサ、データ線に直列抵抗を最初から部品リストへ。
KiCadのチュートリアル通りに操作できないKiCadのバージョン差(8系/9系)でUIが違う。チームでバージョンを揃え、そのバージョンのdocsを見る。
KiCadのライブラリに部品が無いモータドライバIC等は未収録が多い。SnapEDA等で入手、無ければシンボル/フットプリントを自作。
安全上の注意（必読）

(1)配線・差し替えは必ず電源を切ってから。通電したまま挿し替えるとショートで部品が壊れる。(2)電池はリチウム系（ショート・過放電・物理的破損で発火の恐れ）。極性を必ず確認し、+と−を直結しない。保護回路付きを選ぶ。(3)ブレッドボードに大電流（モーター）を流さない（発熱・発火）。(4)最初は低い電圧/輝度で試し、テスターで電圧を確かめてから本番電源につなぐ。(5)はんだ付け（Phase2以降）は換気とやけどに注意。

動作確認のしかた

Phase1（ブレッドボード）で見るべきこと:

  1. テスターの導通モードで GND同士・各電源ラインがつながっているか確認（ショートしていないかも）。
  2. テスターのDCVモードで各電源電圧（電池電圧・5V・Pi5の3.3V）が想定どおりか確認。
  3. モータ制御の最小テストコードでモーターが正転・逆転するか。
  4. NeoPixelの最小テストコードでLEDが指定した色で正しく光るか（色化け・チラつきが無いか）。
  5. 確定した配線を写真or Fritzingで残せているか（配線図がある＝Phase1のDone）。

Phase2（KiCad）で見るべきこと:

  1. 回路図(schematic)がブレッドボードの確定配線と一致しているか。
  2. 全部品にフットプリントが割り当たっているか。
  3. PCBエディタで未配線(ratsnestの白線)が残っていないか。
  4. DRC（設計ルールチェック）がエラー0・違反0で通るか。
  5. 製造データ（ガーバー＋ドリルファイル）が出力できたか（＝このIssueのゴール達成）。

用語ミニ辞典

言葉かんたんな意味
ブレッドボードはんだ付けせず部品を挿して仮配線できる実験ボード。大電流(目安1A)に弱い。
テスター(マルチメータ)電圧・導通(つながっているか)などを測る道具。黒=COM/赤=VΩ端子に挿し、導通モード/DCVモードを切替えて使う。
RP1チップRaspberry Pi 5 で新搭載のGPIO等を担当するチップ。これのせいで旧来のNeoPixel駆動方法が使えないことがある。
NeoPixel(WS281x)1線で多数つなげるフルカラーLED。1ピクセル最大約60mA。データは5Vロジック。
レベル変換(レベルシフタ)電圧の違う信号(3.3V↔5V)をつなぐ部品。74AHCT125等。Pi(3.3V)→NeoPixel(5V)では基本必要。
デカップリングコンデンサ電源を安定させ突入電流を吸収するコンデンサ。NeoPixel電源に数百〜1000µFが定番。
直列抵抗(データ線)NeoPixelのデータ線に入れる約330〜470Ωの抵抗。信号を保護する定番。
モータドライバPi5の弱い信号で大電流のモーターを回すための中継部品。
GND共通化全回路の基準(0V)をつなぐこと。これが無いと信号が正しく伝わらない。
フットプリントKiCadで部品の実物サイズ(はんだ付けする穴/パッドの形)を表すデータ。
DRC(設計ルールチェック)KiCadが配線間隔や未配線などの違反を自動で検出する機能。
ratsnest(ラッツネスト)PCBで『まだ実配線していないつながり』を示す細い白線。
ガーバー基板を製造するための標準データ形式。これを工場に渡すと基板ができる。
