🎯 このアプリでやりたいこと

利用者がスマホ／PCのアプリで本を探し、「自席まで本棚ごとロボに呼び出す」ことができるシステムです。

  • 本を 探す・借りる・返す
  • 本棚ロボを 自分の席に呼ぶ
  • 貸出の 履歴を残す

使う技術： Next.js（画面）＋ Supabase（データの保管庫）＋ Vercel（公開）。ロボ側は Raspberry Pi + ROS2 で別管理。

📖 用語集（初心者向け）

リポジトリ
プログラムや作業の履歴をまとめて保管する「箱」。Gitで管理します。
Next.js（ネクストジェイエス）
Webサイト・アプリの「画面」を作るための道具（フレームワーク）。
Supabase（スーパーベース）
データを保管しておく「クラウド上のデータベース」。本の情報や貸出履歴をここに入れます。
テーブル（表）
Excelの表のような、データの入れ物。「本の表」「貸出の表」のように種類ごとに分かれています。
スキーマ
「どんな表を、どんな列で作るか」を書いた設計図のこと。
publishable key（鍵）
アプリがSupabaseに接続するための「合言葉」。公開しても安全な種類の鍵です（※ secret key は危険なので共有しません）。
.env.local
接続先URLや鍵など、秘密の設定値を書いておくファイル。GitHubには上げない設定になっています。
npm install
アプリの動作に必要な「部品」をまとめてダウンロードするコマンド。
